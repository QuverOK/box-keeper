# BoxKeeper

Веб-приложение для учёта вещей в коробках и их размещения в виртуальных хранилищах (комнатах, кладовых, гаражах). Помогает не потерять, где что лежит: коробки на плане, предметы внутри коробок, поиск и QR-метки.

**Стек:** React + Vite (frontend), NestJS + Prisma + PostgreSQL (backend).

---

## Что умеет приложение

| Возможность | Описание |
|-------------|----------|
| **Регистрация и вход** | Личный аккаунт, JWT-авторизация |
| **Хранилища** | Несколько комнат с заданными размерами (ширина × глубина × высота) |
| **Коробки** | Создание, цвет, размеры, размещение на плане drag-and-drop |
| **Предметы** | Название, категория, описание, фото — внутри каждой коробки |
| **План и 3D** | 2D-план сверху и 3D-вид хранилища с коробками |
| **Перегородки и метки** | Препятствия на плане и текстовые подписи зон |
| **Поиск** | По имени коробки, вещи, категории или описанию |
| **QR-коды** | У каждой коробки уникальный код; сканер открывает содержимое |
| **Гостевой просмотр** | По QR можно посмотреть коробку без входа (режим только чтения) |
| **Профиль** | Смена имени и пароля |

## Быстрый старт (Docker)

На машине нужен только **Docker** (Desktop или Engine + Compose v2). Node.js, PostgreSQL и npm ставить **не обязательно**.

### 1. Клонировать репозиторий

```bash
git clone https://github.com/QuverOK/box-keeper.git
cd box-keeper
```

### 2. Настроить зеркало Docker Hub (обязательно)

В ряде сетей Docker Hub недоступен или отвечает с таймаутом (`TLS handshake timeout`). Перед первым запуском настройте зеркало:

1. Откройте **Docker Desktop** → **Settings** → **Docker Engine**.
2. Добавьте `registry-mirrors` в JSON. Если другие поля уже есть — **не удаляйте их**, только допишите:

```json
{
  "registry-mirrors": [
    "https://dh-mirror.gitverse.ru",
    "https://dockerhub.timeweb.cloud",
    "https://dockerhub1.beget.com"
  ]
}
```

3. Нажмите **Apply & restart**.

Проверка:

```bash
docker info
```

В выводе должна быть секция **Registry Mirrors**. Затем:

```bash
docker pull postgres:16-alpine
```

Если образ скачался — можно запускать проект.

> **Безопасность:** публичные зеркала — сторонние серверы. Для учебного использования это обычно приемлемо; для продакшена лучше своё зеркало или VPN.

### 3. Запустить приложение

Из корня репозитория:

```bash
npm run docker:up
```

Первый запуск занимает несколько минут: скачиваются образы, собираются frontend и backend, применяются миграции БД и загружаются демо-данные.

### 4. Открыть в браузере

**http://localhost:3000**

### 5. Войти

После первого запуска в БД создаётся демо-пользователь:

| Поле | Значение |
|------|----------|
| Email | `test@boxkeeper.local` |
| Пароль | `Test1234!` |

### 6. Остановить

```bash
npm run docker:down
```

## Режимы Docker

### Production (рекомендуется для первого знакомства)

```bash
npm run docker:up
```

- Frontend: **http://localhost:3000** (nginx + статическая сборка)
- Backend, PostgreSQL — внутри Docker-сети
- При старте: миграции + seed (если `RUN_SEED=true`)

### Dev (разработка с hot reload)

```bash
npm run docker:dev
```

| Сервис | Адрес |
|--------|-------|
| Frontend | **https://localhost:3000** (самоподписанный SSL — примите предупреждение в браузере) |
| Backend | http://localhost:4000 |
| PostgreSQL | localhost:5432 |

Остановка:

```bash
npm run docker:dev:down
```

## Настройки (опционально)

Скопируйте `.env.docker.example` в `.env` в **корне** репозитория:

```bash
cp .env.docker.example .env
```

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `JWT_SECRET` | `change-me-in-production` | Секрет для JWT-токенов |
| `JWT_EXPIRES_IN` | `7d` | Срок жизни токена |
| `RUN_SEED` | `true` | Заполнять БД демо-данными при каждом старте backend |
| `FRONTEND_PORT` | `3000` | Порт frontend на хосте |

После первого успешного запуска можно поставить `RUN_SEED=false`, чтобы seed не перезаписывал демо-пользователя при рестарте контейнеров.

## Как пользоваться приложением

### Главный экран (Dashboard)

После входа отображается список ваших **хранилищ**. Можно создать новое (укажите название и размеры комнаты в метрах) или открыть существующее.

В демо-данных уже есть склады, например **«Демо: поиск и UI»**, **«Стек 50 коробок»** и другие.

### Редактор хранилища

Откройте склад — откроется редактор с:

- **2D-план** — вид сверху, перетаскивание коробок
- **3D-вид** — объёмное представление комнаты
- **Режим редактирования** — добавление коробок, перегородок, текстовых меток, изменение размеров комнаты
- **Поиск** — строка вверху; ищет по коробкам и вещам, подсвечивает результат на плане

Примеры поиска в демо-складе «Демо: поиск и UI»:

| Запрос | Найдёт |
|--------|--------|
| `alpha` | коробку Alpha |
| `отвёртка` | вещь в Beta-инструментах |
| `инструменты` | по категории |
| `архивный` / `2024` | по описанию в Gamma-документах |

### Коробка и вещи

- Клик по коробке на плане → карточка коробки со списком **предметов**
- Можно добавить, изменить или удалить вещи
- У коробки есть **QR-код** — кнопка показа/скачивания

### QR-сканер

- На экране входа или в меню — **сканирование QR**
- Наведите камеру на QR коробки (в dev-режиме нужен HTTPS и разрешение камеры)
- Демо-код для теста: `SEED-QR-DEMO`
- Без входа откроется **гостевой просмотр** коробки (только чтение)

### Профиль

Раздел **Профиль** — смена имени и пароля.

### Регистрация

Можно создать свой аккаунт через форму регистрации на экране входа. Новый пользователь начинает с пустого списка хранилищ.

## Архитектура в Docker

```
Браузер → http://localhost:3000 (frontend / nginx)
              │
              │  /api/*
              ▼
         backend:4000 (NestJS)
              │
              ▼
         db:5432 (PostgreSQL, база boxkeeper)
```

Контейнеры:

| Сервис | Образ / сборка | Назначение |
|--------|----------------|------------|
| `db` | `postgres:16-alpine` | База данных |
| `backend` | `backend/Dockerfile` | API, миграции, seed |
| `frontend` | `frontend/Dockerfile` | UI, прокси `/api` → backend |

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| `TLS handshake timeout` при pull | Настройте `registry-mirrors` (см. выше) |
| `tsx: not found` / ошибка seed | Пересоберите: `docker compose build --no-cache backend` |
| Пустой экран / ошибки API | Дождитесь логов backend: `Backend running on http://localhost:4000/api` |
| Порт 3000 занят | В `.env` задайте `FRONTEND_PORT=3001` |
| Камера QR не работает (dev) | Откройте **https://localhost:3000**, разрешите камеру |
| Старый кэш образов | `docker compose build --no-cache && docker compose up` |

Просмотр логов:

```bash
docker compose logs -f backend
docker compose logs -f frontend
```
